1. Diagnóstico y funciones del entorno informático

El diagnóstico informático comprende la evaluación de hardware, software y redes para detectar fallos y optimizar el rendimiento.

Las herramientas de diagnóstico incluyen software de monitoreo de hardware, redes y aplicaciones, como Nagios, Wireshark y Nmap, que permiten identificar problemas y analizar el comportamiento del sistema en tiempo real.

Estas herramientas facilitan la realización de análisis en contextos reales, permitiendo a los técnicos anticipar fallos y mejorar la infraestructura IT.

Fuentes:

  • Herramientas de diagnóstico de red: Wireshark, Nagios, Nmap, Datadog (InvGate Blog, 2022)
  • Software de monitoreo para infraestructura IT (Cube, 2022)
  • Diagnóstico para transformación digital (UNL, 2024)

2. Autogestión y operaciones comerciales

2.1 Actividades de apoyo a la compra y venta

Incluye la gestión personal de tareas vinculadas al circuito comercial, como seguimiento de pedidos, control de inventarios y atención al cliente.

2.2 El mercado informático

  • Las fuentes de aprovisionamiento pueden ser institucionales o a través de medios de comunicación especializados
  • Las modalidades de trabajo varían desde la venta directa hasta contratos de servicio
  • La determinación del precio de venta considera costos, demanda y competencia
  • El Impuesto al Valor Agregado (IVA) es un tributo que afecta el precio final y debe ser gestionado adecuadamente
  • Las operaciones comerciales incluyen intereses implícitos, como descuentos o financiamiento

2.3 Usos y costumbres comerciales

  • Las operaciones habituales incluyen emisión de facturas, remitos y otros documentos legales
  • Es fundamental cumplir con las obligaciones fiscales y legales para evitar sanciones

Fuentes:

  • Diagnóstico empresarial y gestión comercial (HubSpot, 2024)
  • Herramientas de análisis empresarial para gestión comercial (Brand24, 2025)

3.1 Matemática financiera básica

3.2 Estadística descriptiva

5. Ética y deontología profesional

Los códigos de ética profesional establecen normas de conducta esperadas en el ámbito informático, promoviendo responsabilidad, confidencialidad y respeto.

La conducta ética es fundamental para la confianza y la integridad en proyectos tecnológicos.

6. Adaptación y desarrollo de programas

6.1 Análisis y especificación de requerimientos

  • Fundamentos del análisis de software para definir claramente las necesidades del usuario
  • La especificación clara de requerimientos es clave para el éxito del proyecto
  • Validación con el usuario para asegurar el acuerdo en objetivos y funcionalidades

6.2 Técnicas de relevamiento

  • Entrevistas abiertas para obtener información detallada
  • Cuestionarios para recopilar datos estructurados
  • Representación gráfica (diagramas, mapas) para visualizar características y problemas

7. Bases de datos

7.1 Fundamentos

  • Arquitecturas comunes incluyen cliente-servidor y distribuidas
  • El modelo relacional organiza datos en tablas con relaciones definidas
  • La normalización evita redundancias y mejora la integridad

7.2 Diseño y resolución

  • Generación de vistas para facilitar consultas específicas
  • Resolución de problemas mediante consultas SQL
  • Lenguajes de consulta estructurados para manipulación y recuperación de datos

8. Diseño de interfaces y generación de informes

Una buena interfaz debe ser intuitiva, accesible y eficiente.

  • Uso de técnicas y herramientas como prototipos, wireframes y software especializado
  • Elaboración de informes adaptados a requerimientos específicos del usuario para facilitar la toma de decisiones

9. Mantenimiento de integridad de datos

9.1 Estructuras de archivos

  • Tipos varían según entorno (secuenciales, indexados, etc.)
  • Representación y almacenamiento eficientes para acceso rápido
  • Identificación y vinculación de datos para mantener coherencia

9.2 Programas utilitarios

  • Herramientas para conversión entre formatos y edición de datos en bruto

9.3 Bases de datos

  • Mantener integridad y coherencia mediante reglas y restricciones
  • Administración de problemas relacionados con corrupción o inconsistencias

10. Seguridad informática

10.1 Virus informáticos

  • Tipos incluyen virus, gusanos, troyanos, ransomware
  • Funcionamiento basado en replicación y daño a sistemas
  • Programas antivirus operan mediante detección y eliminación, con actualizaciones constantes
  • Prevención mediante buenas prácticas y reparación tras infección

10.2 Encriptado de datos

  • Aplicado a archivos, transacciones y mensajes para proteger confidencialidad
  • Métodos incluyen cifrado simétrico y asimétrico, con herramientas como SSL/TLS

10.3 Protección y control de acceso

Principios de restricción para limitar acceso solo a usuarios autorizados.

  • Métodos incluyen autenticación, autorización y auditoría
