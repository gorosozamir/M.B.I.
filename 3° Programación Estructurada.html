1. Programación Orientada a Objetos (POO)

Fundamentos de la Programación Orientada a Objetos

La POO es un paradigma de programación basado en la estructura de objetos, que contienen datos (atributos) y funciones (métodos) que manipulan dichos datos. Se orienta a modelar entidades del mundo real y sus interacciones para facilitar la organización, reutilización y mantenimiento del código.

Relación entre la POO y la calidad del software: POO contribuye a mejorar la calidad del software al promover modularidad, reutilización de código, abstracción y encapsulamiento, lo cual aumenta la mantenibilidad, reduce errores y facilita la evolución del software.

Conceptos Clave de POO

  • Clases y objetos: Una clase es un molde o plantilla que define atributos y comportamientos comunes; un objeto es una instancia concreta de una clase.
  • Encapsulamiento: Oculta la información interna del objeto, permitiendo acceso solo a través de métodos públicos, protegiendo datos y facilitando la modularidad.
  • Herencia: Permite que una clase derive características y comportamientos de otra, favoreciendo la reutilización y extensión del código.
  • Polimorfismo: Permite que diferentes clases respondan a la misma interfaz de manera distinta, facilitando la flexibilidad y el diseño extensible.

Contextualización de estructuras de datos en POO

Las estructuras de datos se modelan como clases u objetos, encapsulando tanto los datos (nodos, elementos) como operaciones (inserción, eliminación), integrándose con los principios de la orientación a objetos para una mejor organización y abstracción.

Beneficios de la POO

  • Modularidad: El código se organiza en componentes independientes (objetos)
  • Reutilización: Las clases pueden ser extendidas y reutilizadas en diferentes contextos
  • Mantenibilidad: Cambios en una parte del sistema tienen menor impacto en otras
  • Escalabilidad: Facilita el crecimiento y evolución del software
  • Modelado realista: Representación más intuitiva de problemas del mundo real

2. Lenguajes de Programación

Definición de un lenguaje algorítmico

Es un lenguaje formal que permite expresar algoritmos de manera estructurada y comprensible para ser ejecutados por computadoras, con reglas sintácticas, semánticas y pragmáticas claras.

Lenguaje Java

Aspectos sintácticos

Utiliza una sintaxis similar a C++, con definición de clases, uso de llaves para bloques, tipos de datos estáticos y una estructura clara de métodos y constructores.

Aspectos semánticos

Establece el significado de las instrucciones, control de tipos, comportamiento de objetos, manejo de memoria automática, y reglas para ejecución correcta.

Aspectos pragmáticos

Incluyen diseño orientado a objetos, portabilidad multiplataforma mediante la JVM, manejo de excepciones, concurrencia y librerías estándar para diversas aplicaciones.

Recursos de Java para:

  • Iteración: Bucles for, while, do-while, y el uso de iteradores para recorrer colecciones.
  • Abstracción: Definición de clases abstractas e interfaces para describir comportamientos sin implementar detalles concretos.
  • Implementación de estructuras de datos: Uso de clases como ArrayList, LinkedList, Stack, y definición personalizada con encapsulamiento y métodos específicos.

3. Tipos y Estructuras de Datos

Conceptos fundamentales

  • Tipos de datos: Son categorías que identifican la naturaleza del dato (int, float, char, boolean, etc.) y definen operaciones válidas.
  • Estructuras de datos: Formas de organizar y almacenar datos para facilitar acceso y modificación (listas, pilas, colas, árboles).
  • Tipos abstractos de datos (TAD): Definiciones de estructuras de datos que incluyen operaciones específicas, independiente de su implementación.
  • Tipos de datos recursivos: Tipos definidos en términos de sí mismos, usados para estructuras como listas enlazadas y árboles.
  • Representación de datos en memoria: Datos almacenados secuencial o enlazadamente, usando direcciones de memoria y punteros para estructuras dinámicas.

Estructuras básicas

Pilas (Stacks)

  • Definición: TAD que modela una colección LIFO (último en entrar, primero en salir).
  • Implementación: Con arreglos o listas enlazadas, operaciones principales: push (insertar), pop (eliminar), y peek (consultar).
  • Aplicaciones: Recursión, reversión y análisis sintáctico.

Colas (Queues)

  • Definición: TAD para una colección FIFO (primero en entrar, primero en salir).
  • Implementación: Similar a pilas, con operaciones enqueue (insertar) y dequeue (eliminar).
  • Variantes: Colas dobles, circulares y de prioridad que combinan comportamientos de pilas y colas.

4. Estructuras Dinámicas

Estructuras enlazadas

Listas donde cada elemento apunta al siguiente (y opcionalmente al anterior), permitiendo inserciones y eliminaciones dinámicas.

Estructuras dinámicas

Permiten modificar su tamaño durante la ejecución, facilitando gestión flexible de memoria y datos.

Aplicaciones

Manejo eficiente de memoria, pilas o colas dinámicas, árboles, grafos.

Representación en POO

Cada nodo y estructura representado como objeto con atributos y métodos, facilitando modularidad y reutilización.

Implementación en Java

Definición de clases para nodos y estructuras, uso de referencias para manejar enlaces.

5. Iteración y Recorridos

Recorrido de estructuras de datos

Técnicas para visitar todos los elementos, ej. recorrido secuencial, preorden/inorden/postorden en árboles.

Mecanismos de abstracción

Uso del TAD Iterador que unifica la interfaz para recorrer distintas estructuras sin conocer su implementación específica.

Java y iteración

La interfaz Iterator permite recorrer colecciones de manera uniforme con métodos hasNext() y next().

Iteración en estructuras lineales

Aplicación de iteradores en listas, pilas y colas para procesar sus elementos.

6. Conjuntos y Tipos Abstractos Derivados

TAD Conjunto

Modelo que representa colecciones sin orden ni duplicados, con operaciones de inserción, eliminación y consulta.

Estructuras para conjuntos

Arrays, listas enlazadas, tablas hash, árboles balanceados para representación eficiente.

Modelos conjuntistas

  • Colas con prioridad: Seleccionan el próximo elemento basado en una prioridad.
  • Diccionarios o mapas: Asocian llaves a valores utilizando estructuras como tablas hash o árboles.

Aplicaciones e implementaciones

Búsqueda rápida, gestión de datos, bases de datos en memoria.

Análisis comparativo

Depende de requisitos de eficiencia en acceso, inserción, eliminación y uso de memoria.

7. Almacenamiento Externo

Medios de almacenamiento externo

Discos duros, SSD, cintas magnéticas, memorias flash, etc., clasificados por tecnología, velocidad y acceso.

Organización

  • Secuencial: Datos almacenados uno tras otro.
  • Con índices: Listas auxiliares para acelerar búsqueda.
  • Acceso directo: Datos accesibles en cualquier orden sin leer secuencialmente.

Archivos

  • Organización secuencial: Usada en listas y archivos de texto.
  • Organización con índices: Para optimizar búsquedas.
  • Acceso directo: Permite actualización rápida sin reescribir todo el archivo.

Estructuras de datos aptas para almacenamiento externo

Árboles B y variantes, listas indexadas.

Aplicaciones prácticas

Bases de datos, sistemas de archivos, almacenamiento en caché.

8. Competencias a Desarrollar

Habilidades del programador

  • Buscar y analizar código ajeno: Para entender su estructura y funcionalidad.
  • Elegir la mejor implementación de TAD: Considerando eficiencia y contexto del problema.
  • Analizar críticamente estructuras de datos: Según flexibilidad, rendimiento, y adaptabilidad.
  • Pensamiento algorítmico: Resolución lógica de problemas.
  • Depuración: Identificación y corrección de errores.
  • Trabajo en equipo: Colaboración en proyectos.
  • Aprendizaje continuo: Adaptación a nuevas tecnologías.
