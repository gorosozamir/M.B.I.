Unidad 1: Fundamentos de Redes de Datos

1.1 Arquitectura de redes

Modelo OSI

Es un modelo de referencia que divide la comunicación en 7 capas:

  • Capa física: Transmisión de bits a través del medio
  • Capa de enlace de datos: Gestión de tramas y control de flujo
  • Capa de red: Enrutamiento y direccionamiento lógico
  • Capa de transporte: Control de conexiones y confiabilidad
  • Capa de sesión: Establecimiento y mantenimiento de sesiones
  • Capa de presentación: Traducción y cifrado de datos
  • Capa de aplicación: Interfaz con aplicaciones de usuario

Concepto de servicio

En redes, un servicio es la función que una capa ofrece a la capa superior, abstraída de cómo se implementa.

1.2 Tipos de redes

LAN (Local Area Network)

Redes de área local que conectan dispositivos en espacios reducidos como oficinas o edificios, usando cables Ethernet o Wi-Fi.

WAN (Wide Area Network)

Redes de área amplia que conectan redes LAN a gran escala geográfica, utilizando tecnologías como MPLS, VPN y enlaces satelitales.

Topologías de red

Disposición física o lógica de los nodos:

  • Estrella: Todos los nodos conectados a un punto central
  • Bus: Todos los nodos conectados a un cable principal
  • Anillo: Nodos conectados en círculo
  • Malla: Conexiones redundantes entre nodos

Técnicas y medios de transmisión

  • Cables de cobre (UTP, STP)
  • Fibra óptica (monomodo, multimodo)
  • Ondas de radio (Wi-Fi, Bluetooth)
  • Infrarrojos

Unidad 2: Capas del Modelo OSI y dispositivos asociados

2.1 Nivel físico

  • Mecanismos de transmisión de bits a través de medios físicos
  • Corrección y detección de errores: técnicas como paridad, CRC
  • Protocolos de nivel físico definen señales eléctricas, ópticas o radio
  • Dispositivos: Hubs (repetidores multi-puerto) y Repetidores (amplifican señales)

2.2 Nivel de enlace de datos

  • Protocolos como Ethernet y Token Ring gestionan la comunicación dentro de una red local
  • Control de acceso al medio (MAC)
  • Detección y corrección de errores
  • Dispositivos: Bridges (filtran tráfico entre segmentos) y Switches (conmutan tramas a puertos específicos)

2.3 Capa de red

  • Protocolos de enrutamiento que determinan la mejor ruta para los paquetes (ej. IP)
  • Algoritmos de ruteo como RIP, OSPF, BGP
  • Fragmentación y ensamblado de paquetes
  • Dispositivos: Routers (dirigen paquetes entre redes) y Firewalls (controlan acceso y seguridad)

2.4 Conmutación

  • Conmutación a nivel 2 (tramas) y nivel 3 (paquetes)
  • Usada para dirigir tráfico en redes
  • Switches gestionan tablas MAC para conmutación eficiente

2.5 Capa de transporte

  • Protocolos como TCP que garantizan la entrega fiable de datos
  • Administración de conexiones y control de flujo
  • Multiplexación y desmultiplexación
  • Control de congestión

Unidad 3: Protocolos WAN y tecnologías inalámbricas

Protocolos WAN

  • X.25: Protocolo antiguo para redes de conmutación de paquetes
  • Frame Relay: Tecnología de conmutación de paquetes más eficiente
  • ISDN: Red Digital de Servicios Integrados (RDSI)
  • ATM: Modo de Transferencia Asíncrona, para voz, video y datos

Tecnologías inalámbricas

  • Wi-Fi: Para redes locales inalámbricas (IEEE 802.11)
  • Bluetooth: Conexiones de corto alcance
  • Protocolos de integración: Permiten interoperabilidad entre diferentes tecnologías

Unidad 4: Cableado estructurado

Tipos y categorías de cables

  • UTP (Unshielded Twisted Pair): Par trenzado no blindado
  • STP (Shielded Twisted Pair): Par trenzado blindado
  • Fibra óptica: Monomodo y multimodo
  • Categorías: Cat5e, Cat6, Cat6a, Cat7

Normas y estándares

  • TIA/EIA-568: Estándar para cableado estructurado
  • ISO/IEC 11801: Estándar internacional
  • Normas de instalación y certificación

Técnicas de armado

  • Crimpado de cables RJ45
  • Montaje de patch panels
  • Pruebas de certificación
  • Organización de racks y gabinetes

Unidad 5: Seguridad en redes

Políticas de seguridad

  • Control de acceso a datos personales, comerciales y profesionales
  • Autenticación, autorización y auditoría
  • Listas de control de acceso (ACL)

Conceptos fundamentales

  • Confidencialidad: Protección contra accesos no autorizados
  • Integridad: Garantía de que los datos no son alterados
  • Disponibilidad: Acceso autorizado cuando se necesita

Mecanismos de seguridad

  • Cifrado de datos (SSL/TLS, IPsec)
  • Firewalls y sistemas IDS/IPS
  • VPNs para acceso remoto seguro
  • Segmentación de redes (VLANs)
